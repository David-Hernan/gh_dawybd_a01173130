    Preguntas a responder

    1. ¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

      2. Para evitar que el servidor almacene información que es errónea ya que esto podría provocar que algunas funciones del mismo fallen. Por ejemplo, no se podrán realizar operaciones si el usuario ingresa información del tipo String cuando se esperaban datos tipo Int.

    2. ¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

      3. Desde la consola, identificando y modificando las variables que afectan dichas validaciones de seguridad o, desactivando JavaScript desde el navegador.

    3. Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

      4. Porque la primera pregunta sirve para impedir la entrada de datos erróneos. Referente a la seguridad, la misma no debe dejarse sólo al nivel de JavaScript pues existen otros métodos que deben implementarse para poder garantizarla.
